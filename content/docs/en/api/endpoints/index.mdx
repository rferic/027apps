# Endpoints

All paths are relative to the base URL `/api/v1`.

## Available endpoints

| Method | Path | Auth | Description |
|---|---|---|---|
| `GET` | `/api/v1` | Public | [Health check](/doc/api) — returns version and status |
| `GET` | [`/api/v1/me`](/doc/api/endpoints/me) | JWT | Authenticated user info, group membership, and profile |
| `GET` | [`/api/v1/apps`](/doc/api/endpoints/apps) | JWT or API Key | Apps available to the group (or filtered by user permissions) |
| `GET` | [`/api/v1/locales`](/doc/api/endpoints/locales) | Public | Active locales configured for the group |

## Authentication levels

- **Public** — No `Authorization` or `X-API-Key` header needed.
- **JWT** — Requires `Authorization: Bearer <token>`. The token must belong to an active group member.
- **API Key** — Requires `X-API-Key: sk_027_...`. The key must be active (not revoked).
- **JWT or API Key** — Either credential type is accepted; the response may differ depending on which is used.

See [Authentication](/doc/api/authentication) for how to obtain credentials.
