# Autenticazione

L'API di 027Apps supporta due tipi di credenziali: **token JWT** emessi da Supabase Auth e **chiavi API** generate dal pannello di amministrazione.

## JWT (token Bearer)

Usa un JWT quando costruisci un client browser o mobile dove un utente è connesso. Il token identifica sia l'utente che il gruppo a cui appartiene.

### Ottenere un token

Accedi tramite la pagina di login dell'applicazione o direttamente con l'API di Supabase Auth. Dopo un accesso riuscito, Supabase restituisce un `access_token`:

```typescript
import { createClient } from '@supabase/supabase-js'

const supabase = createClient(SUPABASE_URL, SUPABASE_ANON_KEY)

const { data, error } = await supabase.auth.signInWithPassword({
  email: 'alice@example.com',
  password: 'secret',
})

const token = data.session?.access_token
```

I token scadono dopo un'ora. Usa `supabase.auth.refreshSession()` o il rinnovo automatico integrato nel client Supabase per ottenerne uno nuovo.

### Usare il token

Passa il token nell'header `Authorization`:

```bash
curl https://example.com/api/v1/me \
  -H "Authorization: Bearer <access_token>"
```

```typescript
const res = await fetch('/api/v1/me', {
  headers: {
    Authorization: `Bearer ${token}`,
  },
})
const user = await res.json()
```

## Chiave API

Usa una chiave API per le integrazioni server-to-server dove non c'è un utente interattivo. Le chiavi API sono di lunga durata e associate al gruppo.

### Ottenere una chiave API

1. Accedi come amministratore.
2. Vai su **Admin → Impostazioni → Chiavi API**.
3. Clicca su **Nuova chiave API**, inserisci un nome e scegli un ambito:
   - **Gruppo** — accesso machine-to-machine; nessun contesto utente.
   - **Utente** — agisce per conto di un utente specifico.
4. Copia la chiave immediatamente. Viene mostrata **una sola volta** e non viene mai memorizzata in testo semplice.

Le chiavi seguono il formato `sk_027_<32 caratteri casuali>` (es. `sk_027_AbCdEfGhIjKlMnOpQrStUvWxYz012345`).

### Usare una chiave API

Passa la chiave nell'header `X-API-Key`:

```bash
curl https://example.com/api/v1/apps \
  -H "X-API-Key: sk_027_AbCdEfGhIjKlMnOpQrStUvWxYz012345"
```

```typescript
const res = await fetch('/api/v1/apps', {
  headers: {
    'X-API-Key': 'sk_027_AbCdEfGhIjKlMnOpQrStUvWxYz012345',
  },
})
const apps = await res.json()
```

### Revocare una chiave API

Vai su **Admin → Impostazioni → Chiavi API** e clicca su **Revoca** accanto alla chiave. Le chiavi revocate vengono rifiutate immediatamente.

## Scegliere tra JWT e chiave API

| Situazione | Usare |
|---|---|
| App mobile o browser, utente connesso | JWT |
| Servizio backend o job pianificato | API Key (ambito gruppo) |
| Agire per conto di un utente specifico senza autenticazione interattiva | API Key (ambito utente) |
| Leggere dati pubblici del gruppo (lingue) | Nessuna autenticazione |

## Risposte di errore

| Scenario | HTTP | Codice `error` |
|---|---|---|
| Nessuna credenziale fornita | 401 | `unauthorized` |
| Il token è scaduto o malformato | 401 | `unauthorized` |
| L'utente non è membro di alcun gruppo | 403 | `forbidden` |
| La chiave API è revocata o sconosciuta | 401 | `unauthorized` |

Consulta [Errori](/doc/api/errors) per il riferimento completo degli errori.
