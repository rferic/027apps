# Endpoints

Todas las rutas son relativas a la URL base `/api/v1`.

## Endpoints disponibles

| Método | Ruta | Auth | Descripción |
|---|---|---|---|
| `GET` | `/api/v1` | Público | [Verificación de estado](/doc/api) — devuelve versión y estado |
| `GET` | [`/api/v1/me`](/doc/api/endpoints/me) | JWT | Información del usuario autenticado, membresía del grupo y perfil |
| `GET` | [`/api/v1/apps`](/doc/api/endpoints/apps) | JWT o API Key | Aplicaciones disponibles para el grupo (o filtradas por permisos de usuario) |
| `GET` | [`/api/v1/locales`](/doc/api/endpoints/locales) | Público | Idiomas activos configurados para el grupo |

## Niveles de autenticación

- **Público** — No se necesita cabecera `Authorization` ni `X-API-Key`.
- **JWT** — Requiere `Authorization: Bearer <token>`. El token debe pertenecer a un miembro activo del grupo.
- **API Key** — Requiere `X-API-Key: sk_027_...`. La clave debe estar activa (no revocada).
- **JWT o API Key** — Se acepta cualquiera de los dos tipos de credencial; la respuesta puede variar según cuál se use.

Consulta [Autenticación](/doc/api/authentication) para saber cómo obtener credenciales.
