## Aperçu

027Apps dispose d'un **Système de Design** personnalisé basé sur la proposition visuelle "Modern".
Tous les nouveaux composants doivent utiliser ce système.

Le système de design se compose de:

- **Design tokens** — propriétés CSS personnalisées pour les couleurs, la typographie, l'espacement et les ombres
- **Composants de base** — composants atomiques réutilisables (Button, Badge, Card, etc.)
- **Composants composites** — composants métier (TodoList, IdeaList, StatCard, etc.)

## Storybook

Un catalogue interactif de composants est disponible via **Storybook**,
où vous pouvez parcourir tous les composants, voir leurs variantes et lire la documentation auto-générée.

```bash
# Démarrer Storybook (serveur de développement sur http://localhost:6006)
pnpm storybook

# Build statique (sortie dans storybook-static/)
pnpm build-storybook
```

## Design Tokens

Tous les tokens sont définis dans `src/design-tokens.css` comme propriétés CSS personnalisées.
Ils sont également reflétés dans le fichier global `src/app/globals.css` sous `:root` et `.dark`.

## Guide d'Utilisation

1. **Ne jamais écrire de HTML/Tailwind brut** si un composant existe dans le système de design
2. **Nouveaux composants** doivent:
   - Être justifiés (pas de duplication)
   - Utiliser les tokens de `design-tokens.css` (jamais de valeurs en dur)
   - Inclure une histoire Storybook avec au moins la variante par défaut + les variantes principales
3. **Même niveau de style**: tous les nouveaux composants doivent utiliser les tokens DS via `var(--token-name)`
4. **Mode sombre**: supporté automatiquement via la classe `.dark` sur le parent

## Ajouter un Nouveau Composant

1. Créer le fichier dans `src/components/ds/nom.tsx`
2. Utiliser les variables CSS de `design-tokens.css` (ne jamais coder les couleurs en dur)
3. Créer `src/components/ds/nom.stories.tsx` avec:
   - Variante par défaut
   - Toutes les variantes principales
   - État interactif si applicable
4. Vérifier: `pnpm build-storybook && pnpm build`
5. Pour les composites métier, utiliser `src/components/composite/`
