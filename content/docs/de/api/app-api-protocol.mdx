# API-Protokoll für Apps

Apps im 027Apps-Ökosystem können ihre eigenen REST-Endpunkte exponieren. Die Plattform verwaltet automatisch Routing, Authentifizierung und Middleware — Sie müssen nur deklarieren, welche Endpunkte Ihre App exponiert und wo sich die Handler-Dateien befinden.

## So deklarieren Sie REST-Endpunkte

### 1. API im Manifest aktivieren

Setzen Sie `"api": true` in der `manifest.json` Ihrer App:

```json
{
  "id": "notes",
  "name": "Notizen",
  "version": "1.0.0",
  "api": true
}
```

### 2. Routen-Dateien erstellen

Platzieren Sie Ihre Route-Handler in `apps/{slug}/routes/`. Die Dateistruktur bestimmt den URL-Pfad:

```
apps/{slug}/routes/
├── route.ts           → GET   /api/v1/admin/apps/{slug}
├── [id]/
│   └── route.ts       → GET   /api/v1/admin/apps/{slug}/:id
│                       → PUT   /api/v1/admin/apps/{slug}/:id
│                       → PATCH /api/v1/admin/apps/{slug}/:id
│                       → DELETE /api/v1/admin/apps/{slug}/:id
```

### 3. HTTP-Methoden exportieren

Jede Routen-Datei exportiert eine oder mehrere von: `GET`, `POST`, `PUT`, `PATCH`, `DELETE`.

```typescript
// apps/notes/routes/route.ts
export async function GET(req: Request, ctx: UseCaseContext) {
  // ...
}

export async function POST(req: Request, ctx: UseCaseContext) {
  // ...
}
```

## Standard-Authentifizierungsstufen

| Stufe | Header | Anwendungsfall |
|---|---|---|
| `public` | Keine | Nicht sensible, gruppenweite Daten |
| `jwt` | `Authorization: Bearer <token>` | Jeder authentifizierte Benutzer |
| `any` | JWT oder `X-API-Key` | Flexibel — unterstützt beide Auth-Methoden |
| `apikey` | `X-API-Key: sk_027_...` | Nur Server-zu-Server-Integrationen |

## Antwortformat

### Erfolg

Ressourcen direkt zurückgeben — ohne äußere Hülle:

```typescript
import { apiOk } from '@/lib/api/response'

// Einzelne Ressource
return apiOk({ id: '123', name: 'Meine Notiz' })

// Erstellte Ressource (201)
return apiOk({ id: '456', name: 'Neue Notiz' }, 201)
```

### Paginierte Listen

```typescript
import { apiOk } from '@/lib/api/response'

return apiOk({
  data: elemente,
  pagination: {
    page: 1,
    limit: 20,
    total: 150,
    total_pages: 8
  }
})
```

### Fehler

```typescript
import { apiError } from '@/lib/api/response'

// Grundlegender Fehler
return apiError('not_found', 'Notiz nicht gefunden', 404)

// Validierungsfehler mit Felddetails
return apiError('validation_failed', 'Eingabe ist ungültig', 400, {
  title: 'Titel ist erforderlich'
})
```

### DELETE-Endpunkte

`204 No Content` ohne Body zurückgeben:

```typescript
import { apiOk } from '@/lib/api/response'

return new Response(null, { status: 204 })
```

### HTTP-Statuscodes

| Code | Bedeutung |
|---|---|
| `200` | OK — erfolgreiches Lesen oder Aktualisieren |
| `201` | Created — erfolgreiche Ressourcenerstellung |
| `204` | No Content — erfolgreiche Löschung |
| `400` | Bad Request — ungültige Eingabe |
| `401` | Unauthorized — fehlende oder ungültige Anmeldedaten |
| `403` | Forbidden — unzureichende Berechtigungen |
| `404` | Not Found — Ressource existiert nicht |
| `500` | Internal Server Error — unerwarteter Fehler |

## Vollständiges Beispiel: Notizen-App

### manifest.json

```json
{
  "id": "notes",
  "name": "Notizen",
  "version": "1.0.0",
  "api": true
}
```

### apps/notes/routes/route.ts (Liste + Erstellen)

```typescript
import { NextRequest } from 'next/server'
import { authenticate } from '@/lib/api/auth'
import { apiOk, apiError } from '@/lib/api/response'
import { createApiAdminClient } from '@/lib/supabase/api'
import type { UseCaseContext } from '@/lib/use-cases/types'

export async function GET(
  req: Request,
  ctx: UseCaseContext
): Promise<Response> {
  // Jeden eingeloggten Benutzer authentifizieren
  const auth = await authenticate(req, 'jwt')
  if (auth instanceof Response) return auth

  const { searchParams } = new URL(req.url)
  const page = Math.max(1, parseInt(searchParams.get('page') ?? '1', 10))
  const limit = Math.min(500, Math.max(1, parseInt(searchParams.get('limit') ?? '20', 10)))
  const offset = (page - 1) * limit

  const admin = createApiAdminClient()

  // Gesamtzahl abrufen
  const { count } = await admin
    .from('notes')
    .select('*', { count: 'exact', head: true })
    .eq('group_id', auth.groupId)

  // Paginierte Elemente abrufen
  const { data: notes } = await admin
    .from('notes')
    .select('id, title, content, created_at, updated_at')
    .eq('group_id', auth.groupId)
    .order('created_at', { ascending: false })
    .range(offset, offset + limit - 1)

  return apiOk({
    data: notes ?? [],
    pagination: {
      page,
      limit,
      total: count ?? 0,
      total_pages: Math.ceil((count ?? 0) / limit)
    }
  })
}

export async function POST(
  req: Request,
  ctx: UseCaseContext
): Promise<Response> {
  const auth = await authenticate(req, 'jwt')
  if (auth instanceof Response) return auth

  const body = await req.json().catch(() => null)
  if (!body || typeof body.title !== 'string') {
    return apiError('validation_failed', 'Titel ist erforderlich', 400, {
      title: 'Eine gültige Titelzeichenfolge ist erforderlich'
    })
  }

  const admin = createApiAdminClient()

  const { data: note, error } = await admin
    .from('notes')
    .insert({
      title: body.title.trim(),
      content: body.content ?? '',
      group_id: auth.groupId,
      created_by: auth.userId
    })
    .select('id, title, content, created_at, updated_at')
    .single()

  if (error) {
    return apiError('server_error', 'Notiz konnte nicht erstellt werden', 500)
  }

  return apiOk(note, 201)
}
```

### apps/notes/routes/[id]/route.ts (Abrufen, Aktualisieren, Löschen)

```typescript
import { authenticate } from '@/lib/api/auth'
import { apiOk, apiError } from '@/lib/api/response'
import { createApiAdminClient } from '@/lib/supabase/api'
import type { UseCaseContext } from '@/lib/use-cases/types'

export async function GET(
  req: Request,
  ctx: UseCaseContext
): Promise<Response> {
  const auth = await authenticate(req, 'jwt')
  if (auth instanceof Response) return auth

  const id = ctx.params?.id
  if (!id) return apiError('not_found', 'Notiz nicht gefunden', 404)

  const admin = createApiAdminClient()
  const { data: note } = await admin
    .from('notes')
    .select('id, title, content, created_at, updated_at')
    .eq('id', id)
    .eq('group_id', auth.groupId)
    .single()

  if (!note) return apiError('not_found', 'Notiz nicht gefunden', 404)

  return apiOk(note)
}

export async function PUT(
  req: Request,
  ctx: UseCaseContext
): Promise<Response> {
  const auth = await authenticate(req, 'jwt')
  if (auth instanceof Response) return auth

  const id = ctx.params?.id
  if (!id) return apiError('not_found', 'Notiz nicht gefunden', 404)

  const body = await req.json().catch(() => null)
  if (!body || typeof body.title !== 'string') {
    return apiError('validation_failed', 'Titel ist erforderlich', 400, {
      title: 'Eine gültige Titelzeichenfolge ist erforderlich'
    })
  }

  const admin = createApiAdminClient()

  // Eigentum prüfen
  const { data: existing } = await admin
    .from('notes')
    .select('id')
    .eq('id', id)
    .eq('group_id', auth.groupId)
    .single()

  if (!existing) return apiError('not_found', 'Notiz nicht gefunden', 404)

  const { data: note, error } = await admin
    .from('notes')
    .update({
      title: body.title.trim(),
      content: body.content ?? '',
      updated_at: new Date().toISOString()
    })
    .eq('id', id)
    .select('id, title, content, created_at, updated_at')
    .single()

  if (error) {
    return apiError('server_error', 'Notiz konnte nicht aktualisiert werden', 500)
  }

  return apiOk(note)
}

export async function DELETE(
  req: Request,
  ctx: UseCaseContext
): Promise<Response> {
  const auth = await authenticate(req, 'jwt')
  if (auth instanceof Response) return auth

  const id = ctx.params?.id
  if (!id) return apiError('not_found', 'Notiz nicht gefunden', 404)

  const admin = createApiAdminClient()

  const { error } = await admin
    .from('notes')
    .delete()
    .eq('id', id)
    .eq('group_id', auth.groupId)

  if (error) {
    return apiError('server_error', 'Notiz konnte nicht gelöscht werden', 500)
  }

  return new Response(null, { status: 204 })
}
```

## Middleware-Konventionen

### Authentifizierungs-Hilfsfunktionen

```typescript
import { authenticate } from '@/lib/api/auth'
import { apiOk, apiError } from '@/lib/api/response'
```

- **`authenticate(req, stufe)`** gibt `UseCaseContext | Response` zurück. Mit `instanceof Response` auf Auth-Fehler prüfen.
- Für Nur-Admin-Endpunkte nach der Authentifizierung `ctx.role === 'admin'` prüfen.

### Datenbankzugriff

```typescript
import { createApiAdminClient } from '@/lib/supabase/api'

const admin = createApiAdminClient()
// Für appspezifische Tabellen verwenden, die nicht in generierten Typen enthalten sind
```

### Paginierung

Paginierung immer für Listen-Endpunkte implementieren:

- `page`: Zahl (Standard `1`, Minimum `1`)
- `limit`: Zahl (Standard `20`, Minimum `1`, Maximum `500`)

### Fehlerbehandlung

```typescript
// Bei Fehlern sofort zurückgeben
const auth = await authenticate(req, 'jwt')
if (auth instanceof Response) return auth

// Immer Code, Nachricht und HTTP-Status übergeben
return apiError('not_found', 'Ressource nicht gefunden', 404)
```
